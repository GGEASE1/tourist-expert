# Обратный проход (Backward Chaining)

## Что имеется в виду под обратным проходом

В рамках продукционной ЭС обратный проход — это вывод **от цели к фактам**:

1. Берем цель (например, выбрать конкретную рекомендацию/правило).
2. Ищем правила, которые могут подтвердить цель.
3. Для каждого кандидата проверяем его условия как подцели.
4. Если все условия правила подтверждаются, цель считается доказанной.
5. Если не доказано ни одно правило, используется fallback на правило по умолчанию.

Это соответствует учебной формулировке из `.misc` (подтверждение цели через правила и цепочку шагов).

## Что реализовано в проекте

### 1) Классический goal-driven backward в движке

Реализовано в `TravelRuleEngine.backward(...)`:

- шаг `prove-goal` — старт доказательства цели;
- шаг `select-rules` — выбор правил-кандидатов;
- шаг `try-rule` — попытка доказать конкретное правило;
- шаг `prove-condition` — проверка каждого условия правила;
- шаги результата:
  - `condition-from-facts` / `condition-failed`,
  - `rule-proved` / `rule-failed`,
  - `goal-proved` / `goal-failed`,
  - `fallback-default` (когда цель `*` не доказана через обычные правила).

Дополнительно возвращаются:

- `matched_rules` — все правила, доказанные в обратном проходе;
- `proof` — структурированное дерево доказательства (кандидаты и их условия);
- `steps` — линейная детальная цепочка шагов.

### 2) Сохранение explain-данных сессии

В сохранение консультации добавлены поля backward-раздела:

- `matched_rules`
- `recommendation`
- `proof`

## Роут `/test`: как показывается цепочка

В тестовом роуте добавлено управление запуском:

- Кнопка **«Запустить тесты»** на странице `/test`.
- До запуска отображается блок ожидания.
- После запуска показываются:
  - результат прямого вывода;
  - результат обратного вывода;
  - **цепочка шагов обратного вывода**;
  - **цепочки по правилам** (по каждому кандидату: доказано/не доказано и статус условий);
  - полный список зарегистрированных правил.

## Прямой вывод

Прямой вывод сохранен и продолжает выполняться отдельно через
`engine.evaluate(..., explain=True)`.
