## 1. Введение

В рамках проекта разрабатывается прототип экспертной продукционной системы «Консультант по путешествиям».  
Основная задача системы — на основе заданных правил и фактов помогать пользователю выбирать подходящее направление для поездки.

Для реализации экспертной системы были рассмотрены две библиотеки:
- ClipSpy
- Experta  

Цель работы — провести их сравнительный анализ и выбрать наиболее подходящую для проекта.

---

## 2. Обоснование выбора языка программирования

В качестве языка разработки был выбран **Python**.

### Причины выбора:

1. **Простота и читаемость**
   - Python легко читается и пишется
   - удобно реализовывать логику правил

2. **Большое количество библиотек**
   - для работы с API (погода, карты и т.д.)
   - для разработки интерфейса
   - для обработки данных

3. **Быстрая разработка**
   - идеально подходит для прототипа
   - меньше времени на написание кода

4. **Поддержка экспертных систем**
   - есть готовые библиотеки (Experta, ClipSpy и др.)

---

### Вывод:
Python является оптимальным выбором, так как он упрощает разработку, поддержку и расширение системы.

---

## 3. Описание рассматриваемых решений

### 3.1 ClipSpy

ClipSpy — это библиотека Python, которая является обёрткой над системой CLIPS.

**Особенности:**
- использует внешний движок CLIPS
- написан на C → высокая скорость
- реализует классическую продукционную систему

**Минусы:**
- сложнее в использовании
- нужно понимать синтаксис CLIPS
- фактически используется два языка (Python + CLIPS)

---

### 3.2 Experta

Experta — это библиотека на Python для создания экспертных систем.

**Особенности:**
- полностью написана на Python
- использует понятные конструкции (Fact, Rule)
- простой синтаксис

**Минусы:**
- ниже производительность
- меньше возможностей по сравнению с CLIPS

---
