# Текущие автоматические тесты проекта

Документ фиксирует актуальное состояние автоматических тестов экспертной системы по состоянию на **5 апреля 2026**.

## Стек и подход

- Тестовый фреймворк: `unittest`.
- Для HTTP-проверок используется `Flask test client`.
- Для изоляции побочных эффектов используется `unittest.mock.patch`.
- Тесты находятся в каталоге `tests/`.

## Состав тестов

Сейчас в проекте **5 тестовых модулей** и **15 тест-кейсов**.

### 1) `tests/test_rules.py` (6 тестов)

Проверяет движок правил `TravelRuleEngine`:

- минимальный объём базы правил;
- прямой вывод для premium-relax сценария;
- fallback-сценарий по умолчанию;
- explain-режим прямого вывода;
- успешный обратный вывод;
- отрицательный сценарий для неизвестной цели.

### 2) `tests/test_form_factory.py` (2 теста)

Проверяет schema-driven генерацию формы:

- порядок видимых полей;
- сборку нормализованных фактов из формы.

### 3) `tests/test_app_route.py` (4 теста)

Проверяет маршруты Flask-приложения:

- `GET /`;
- `POST /`;
- `GET /test`;
- `GET /test?run_tests=1`.

### 4) `tests/test_session_store.py` (1 тест)

Проверяет сохранение консультации в JSON.

### 5) `tests/test_prototype_testing.py` (2 теста)

Проверяет новый демонстрационный слой для `/test`:

- наличие двух сценариев и выполнение требований по шагам/проходам;
- выбор ожидаемых правил в прямом и обратном выводе.

## Актуальная команда запуска

```bash
./.venv/bin/python -m unittest discover -s tests -v
```

## Текущий результат запуска

- Всего тестов: **15**
- Статус: **OK**

## Что покрыто особенно важно

После переработки `/test` автоматическими тестами зафиксировано, что:

- на странице есть два демонстрационных сценария;
- для них строятся и прямой, и обратный вывод;
- сохраняется детальная explain-логика;
- выполняются требования по минимальной длине трассировки;
- не ломается основной маршрут консультации `/`.
