# Текущие тесты проекта

Документ фиксирует актуальное состояние автоматических тестов экспертной системы по состоянию на **4 апреля 2026**.

## Стек и подход

- Тестовый фреймворк: `unittest` (стандартная библиотека Python).
- Для HTTP-проверок используется `Flask test client`.
- Для изоляции побочных эффектов используется `unittest.mock.patch`.
- Тесты расположены в каталоге `tests/`.

## Состав текущих тестов

Всего в проекте 4 тестовых модуля и 13 тест-кейсов.

### 1) `tests/test_rules.py` (6 тестов)

Проверяет модуль продукционных правил `TravelRuleEngine`:

- минимальный объём базы правил (`rules_count() >= 50`);
- прямой вывод для сценария premium-relax (`warm-relax-premium`);
- fallback-сценарий по умолчанию;
- explain-режим прямого вывода (`matched_rules`, `selected_rule`, `steps`);
- успешный обратный вывод (`goal="*"`, построение proof и шагов);
- неуспешный обратный вывод для неизвестной цели (`goal-not-found`).

### 2) `tests/test_form_factory.py` (2 теста)

Проверяет schema-driven генерацию формы и подготовку фактов:

- порядок видимых полей формы соответствует `TRAVEL_FACTS`;
- `get_evaluation_input(form)` собирает нормализованный payload для движка правил.

### 3) `tests/test_app_route.py` (4 теста)

Проверяет маршруты Flask-приложения:

- `GET /` возвращает страницу с заголовком системы;
- `POST /` формирует рекомендацию и вызывает сохранение сессии;
- `GET /test` отрисовывает тестовый экран до запуска;
- `GET /test?run_tests=1` показывает результаты прямого и обратного вывода, цепочки и список правил.

### 4) `tests/test_session_store.py` (1 тест)

Проверяет сохранение консультации:

- `save_consultation_session(payload)` создаёт JSON-файл и корректно записывает ключевые поля.

## Текущий статус запуска

Актуальная команда запуска:

```bash
venv/bin/python -m unittest discover -s tests -v
```
